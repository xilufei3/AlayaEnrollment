# 南科大本科招生智能体前端改版设计

**日期：** 2026-03-19

## 目标

将现有基于 LangGraph 模板的聊天页面，重塑为更贴近“南方科技大学本科招生咨询入口”的前端体验。
本次改版只调整前端表现层，不改动现有聊天请求、线程管理和后端接口行为。

## 当前问题

- 首页仍保留 `Agent Chat`、`Agent Inbox`、`LangGraph` 等模板品牌信息。
- 顶部 `GitHub` 外链和部分开发者向控件不适合招生场景。
- 首屏缺少学校品牌感，像开发调试页，不像正式咨询入口。
- 欢迎区信息层次弱，没有把“我能问什么”清晰传达给考生和家长。
- 配色、排版和留白偏通用，缺少高校招生场景应有的稳重感与可信感。

## 设计原则

- 先做“可信、清晰、正式”，再做装饰性视觉。
- 维持聊天作为核心能力，不额外增加复杂导航和内容层级。
- 保持桌面端和移动端都可直接使用，避免只在大屏好看。
- 预留南科大 Logo 位置，但不硬编码非正式素材。

## 视觉方向

- 整体风格采用“高校招生工作台”而不是“技术产品控制台”。
- 主色以深青绿色、暖白色、浅金色点缀为核心，营造稳重、理性、学术感。
- 顶部与首屏加入柔和渐变、浅纹理和卡片层次，提升品牌识别度。
- 字体继续沿用项目现有 Google Font 机制，但通过字号、粗细和字距拉开信息层次。

## 页面结构

### 1. 顶部品牌栏

- 左侧放置南科大品牌占位徽标和正式标题。
- 标题文案调整为“南方科技大学本科招生智能体”。
- 辅助文案说明用途，例如“面向高中生与家长的招生咨询助手”。
- 删除 GitHub 外链和模板品牌入口。

### 2. 首屏欢迎区

- 首次进入且尚未开始会话时，展示更完整的欢迎区。
- 欢迎区包含：
  - 主标题
  - 一段简短说明，强调可咨询学校概况、综合评价、专业、校园生活等问题
  - 3 到 6 个推荐问题卡片，点击后可快速填入输入框
  - 一组信任信息，例如“基于招生资料整理”“具体政策以官方通知为准”

### 3. 聊天工作区

- 保留现有消息流、滚动到底部、停止生成等核心交互。
- 输入区改为更完整的工作台卡片样式，弱化开发测试感。
- 将“Hide Tool Calls”这类偏开发调试的开关降低存在感，或在视觉上弱化。

### 4. 历史会话区

- 保留历史线程能力，但更换标题和样式，使其更像“咨询记录”。
- 桌面端作为左侧边栏，移动端继续抽屉呈现。
- 历史项样式更简洁，避免和主内容抢视觉焦点。

### 5. 配置缺失页

- 当缺少 API 地址或助手 ID 时，不再展示 LangGraph 技术说明。
- 文案改为项目内部部署说明风格，减少对终端用户的技术暴露。
- 若部署环境已提供默认值，普通访问者基本不会见到这页。

## 保留与删减

### 保留

- 现有线程管理能力
- 消息渲染与工具调用流
- 输入、发送、取消和重新生成能力
- 移动端抽屉式历史记录

### 删减或替换

- 移除 GitHub 外链
- 移除 `Agent Chat`、`Agent Inbox`、`LangGraph` 相关露出
- 移除或弱化明显偏开发调试的提示文案

## Logo 预留方案

- 新增一个简洁的品牌徽标组件，默认使用抽象化的圆角方形或校徽占位图形。
- 组件结构保持可替换，后续拿到正式 `SVG/PNG` 后只需替换图形内容，不影响页面布局。

## 实施范围

主要修改以下区域：

- `web/src/app/layout.tsx`
- `web/src/app/globals.css`
- `web/src/components/thread/index.tsx`
- `web/src/components/thread/history/index.tsx`
- `web/src/providers/Stream.tsx`
- `web/src/components/icons/*`（新增品牌占位图标）

## 验收标准

- 页面不再出现 `Agent Chat`、`Agent Inbox`、GitHub 外链等模板元素。
- 首屏具备明确的南科大招生智能体品牌信息与推荐提问入口。
- 整体视觉较当前版本更正式、统一、适合对外展示。
- 聊天、历史记录、移动端使用不回归。
- 无正式 logo 素材时，页面也具备完整且协调的品牌呈现。
